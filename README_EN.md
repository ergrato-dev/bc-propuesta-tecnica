## 📋 Description

A **9-week (~1 trimester)** bootcamp focused on **Technical Proposal Drafting and Estimation** for software development projects. Designed to take SENA apprentices from zero to drafting and defending a complete professional technical proposal.

Available in **two versions** based on the learner's profile:

- **Open Enrollment (Oferta Abierta)**: 4th trimester of the ADSO Technologist program — prior program knowledge already accumulated
- **Training Chain (Cadena de Formación)**: 3rd-4th trimester — Programming Technician graduates continuing toward the Technologist degree

### 🎯 Objectives

By the end of the bootcamp, apprentices will be able to:

- ✅ Gather functional and non-functional requirements from a client
- ✅ Define the scope of a software project with proper justification
- ✅ Apply at least 3 effort and cost estimation techniques
- ✅ Build a basic project schedule with milestones and critical path
- ✅ Draft a complete professional technical proposal
- ✅ Prepare a commercial proposal with a justified cost table
- ✅ Manage project risks using a probability/impact matrix
- ✅ Present and defend the proposal before an evaluating committee

### 💡 Why this bootcamp?

> **The technical proposal is built section by section from week 1** — by week 9, the final document is nearly complete.

Many software development training programs teach how to code but not how to **communicate and justify** what will be built. This bootcamp fills exactly that gap: a fictional case study is used to learn the tools, while each apprentice's real project serves as the thread connecting all deliverables.

## 🗓️ Bootcamp Structure

### Version A — Open Enrollment (4th Trimester · ADSO)

| Phase | Weeks | Hours | Focus |
| :---: | :---: | :---: | --- |
| **Foundations** | 1-2 | 12h | Advanced requirements analysis + scope and technical feasibility |
| **Estimation** | 3-5 | 18h | Comparative techniques, quantitative estimation, budget |
| **Planning** | 6 | 6h | Schedule, Gantt, CPM, resource management |
| **Proposal** | 7-8 | 12h | Technical drafting + commercial proposal, risks and legal basics |
| **Closing** | 9 | 6h | Presentation and defense — Final Project |

### Version B — Training Chain (3rd-4th Trimester · Programming Technician)

| Phase | Weeks | Hours | Focus |
| :---: | :---: | :---: | --- |
| **Foundations** | 1-3 | 18h | Project context + requirements + basic scope |
| **Estimation** | 4-6 | 18h | Simplified techniques + effort/time + costs |
| **Proposal** | 7-8 | 12h | Technical drafting + commercial proposal and risks |
| **Closing** | 9 | 6h | Presentation and defense — Final Project |

**Total: 9 weeks** | **54 hours** of training

- Active **GitHub** account

## 📊 Learning Methodology

### Teaching Strategies

- 🎯 **Project-Based Learning (PBL)**: the apprentice's real project is the guiding thread
- 🎭 **Client-Consultant Roleplay**: simulated client interviews from week 2
- 🧩 **Incremental construction**: the proposal is built section by section each week
- 👥 **Peer Review**: proposal reviews between apprentices in the final week
- 📚 **Shared case study**: same fictional context for the entire class

### Time Distribution (6h/week)

- **Theory**: 1.5-2 hours
- **Practices**: 2.5-3 hours
- **Project**: 1.5-2 hours

### Evaluation

Each week includes three types of evidence:

1. **Knowledge 🧠** (30%): Quizzes, concept maps, reflection questions
2. **Performance 💪** (40%): Workshops on the fictional case study
3. **Product 📦** (30%): Deliverable applied to the apprentice's real project

**Passing criterion**: Minimum 70% in each type of evidence
