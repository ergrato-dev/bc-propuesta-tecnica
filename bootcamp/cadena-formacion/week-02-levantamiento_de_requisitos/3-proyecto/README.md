# 🚀 Proyecto Semana 2 — Lista de Requisitos de tu Proyecto

> Entregable real · Cadena de Formación · Semana 2 de 9

---

## ¿Qué es este entregable?

Es la **segunda sección de tu propuesta técnica**. Esta semana documentas los requisitos de tu proyecto real: qué debe hacer el sistema (RF) y cómo debe comportarse (RNF).

Al final del bootcamp, esta sección irá integrada en el documento de propuesta técnica completo junto con el entregable de la semana 1.

---

## 📋 Instrucciones

1. **Habla con tu cliente o usuario antes de completar la plantilla.** Si no has tenido esa conversación aún, este es el momento. Usa el guion de entrevista de la teoría (`03-entrevista-cliente.md`) como guía.

2. Abre `starter/entregable-s02-requisitos.md`

3. Lee cada sección y el comentario de instrucciones

4. Completa el documento con la información de tu proyecto real, usando la información que levantaste con tu cliente o usuario

5. Guarda el archivo en tu repositorio personal de GitHub como `entregable-s02-requisitos.md`

6. Comparte el enlace con tu instructor antes de la fecha límite

> ⚠️ **No uses el caso FerreMax aquí.** Ese caso es solo para la teoría y las prácticas. Este entregable es sobre **tu proyecto real**.

---

## ❓ Preguntas Guía

Antes de abrir la plantilla, responde mentalmente estas preguntas:

- ¿Qué acciones concretas podrá hacer el usuario en el sistema? (RF)
- ¿Cómo debe comportarse el sistema en términos de velocidad, seguridad o usabilidad? (RNF)
- ¿Hay restricciones de acceso? ¿Quién puede hacer qué?
- ¿Hay funcionalidades que el cliente mencionó que no están completamente claras aún?
- ¿Tienes evidencia de que hablaste con el cliente? (acta, notas, correo)

---

## 🎤 Si no has hablado con tu cliente todavía...

Esta semana es la ideal para tener la primera reunión formal con el usuario de tu proyecto. Sigue estos pasos:

1. Coordina una reunión de 30-45 minutos (presencial o por Meet/Zoom)
2. Usa el guion de entrevista de `1-teoria/03-entrevista-cliente.md` — adáptalo a tu proyecto
3. Toma notas o graba la reunión (con permiso del cliente)
4. Elabora el acta de reunión inmediatamente después
5. Envíasela al cliente para que confirme lo que se habló

> 💡 La evidencia del levantamiento (acta firmada, correo de confirmación, notas de la reunión) es parte del entregable evaluado.

## ✅ Criterios de Evaluación

Tu entregable será evaluado según la rúbrica `rubrica-evaluacion.md`. Enfócate en:

- Usar el formato correcto: ID único, descripción en "El sistema debe...", prioridad
- Documentar mínimo **8 requisitos funcionales** y **4 no funcionales**
- Incluir evidencia de que levantaste los requisitos con el cliente (acta, notas, correo)
- Que los requisitos sean verificables (evita "el sistema debe ser bueno / rápido / fácil")
- Que los requisitos sean coherentes con el contexto que describiste en el entregable S01
