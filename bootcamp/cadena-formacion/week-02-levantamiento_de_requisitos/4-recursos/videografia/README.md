## Requisitos de Software

### 1. ¿Qué son los Requisitos Funcionales y No Funcionales?
- **URL:** https://www.youtube.com/results?search_query=requisitos+funcionales+no+funcionales+software+explicacion
- **Duración:** ~10-15 min — filtra por menos de 15 min
- **Descripción:** Busca este término en YouTube (Brave/Firefox). Hay varios videos de instructores latinoamericanos que explican la diferencia con ejemplos reales de sistemas. Prioriza los que usen tablas o ejemplos concretos.
- **Cuándo verlo:** Antes o después de leer `01-que-son-los-requisitos.md`

### 2. Cómo redactar Requisitos de Software — ejemplos prácticos
- **URL:** https://www.youtube.com/results?search_query=como+redactar+requisitos+de+software+ejemplos
- **Duración:** ~10 min
- **Descripción:** Videos con ejemplos de cómo pasar de lo que dice el cliente a un requisito bien escrito. Busca ejemplos con tablas RF/RNF.
- **Cuándo verlo:** Junto con la sección de características del buen requisito

## Técnicas de Levantamiento

### 3. Técnicas de Elicitación de Requisitos — Análisis Comparado
- **URL:** https://www.youtube.com/results?search_query=tecnicas+elicitacion+requisitos+entrevista+cuestionario
- **Duración:** ~15 min
- **Descripción:** Busca comparativas entre entrevista, cuestionario y observación aplicadas a proyectos de software. Hay buenos videos de cátedra universitaria latinoamericana.
- **Cuándo verlo:** Junto con `02-tecnicas-levantamiento.md`

### 4. Cómo hacer una Entrevista con el Cliente — Consultoría TI
- **URL:** https://www.youtube.com/results?search_query=entrevista+cliente+levantamiento+requisitos+software+colombia
- **Duración:** ~10-20 min
- **Descripción:** Busca este término — hay freelancers y consultores colombianos que graban sus reuniones con clientes (versión simulada) y luego analizan qué salió bien y qué no. Es el complemento perfecto de `03-entrevista-cliente.md`.
- **Cuándo verlo:** Antes del taller de práctica

## Agile y Requisitos: User Stories

### 5. ¿Qué es una User Story? — Diferencia con Requisito Tradicional
- **URL:** https://www.youtube.com/results?search_query=user+story+historia+usuario+agile+scrum+explicacion+espanol
- **Duración:** ~8 min
- **Descripción:** En metodologías ágiles los requisitos se expresan como "historias de usuario". Este video explica la diferencia con el formato RF-001 tradicional. Relevante para proyectos que usen Scrum o Kanban.
- **Nota:** En este bootcamp usamos el formato RF/RNF tradicional — las user stories son un tema complementario
