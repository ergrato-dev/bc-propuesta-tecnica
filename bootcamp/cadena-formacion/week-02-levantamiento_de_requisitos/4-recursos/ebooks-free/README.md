## Requisitos de Software

### 1. SWEBOK — Capítulo de Requisitos (IEEE)
- **URL:** https://www.computer.org/education/bodies-of-knowledge/software-engineering
- **Descripción:** Cuerpo de conocimiento de ingeniería de software del IEEE. El capítulo 1 (Software Requirements) define formalmente qué son los requisitos funcionales, no funcionales y su proceso de levantamiento.
- **Idioma:** Inglés (referencia técnica)
- **Nivel:** Intermedio — sirve como referencia al escribir los requisitos de la propuesta
- **Qué leer:** Solo el resumen del capítulo 1 (páginas 1-1 a 1-4)

### 2. IEEE 830-1998 — Especificación de Requisitos de Software (resumen)
- **URL:** https://standards.ieee.org/ieee/830/1222/
- **Descripción:** El estándar formal para documentar requisitos. Muchas empresas colombianas lo exigen. Lee el resumen ejecutivo gratuito para entender la estructura estándar.
- **Idioma:** Inglés
- **Nivel:** Referencia técnica — no es lectura completa, solo conocimiento de su existencia

### 3. Ingeniería de Software — Pressman (capítulo de requisitos, resumen libre)
- **URL:** https://www.academia.edu/search?q=pressman+ingenieria+de+software+requisitos
- **Descripción:** Busca en Academia.edu resúmenes del capítulo de requisitos de Pressman. Hay versiones del capítulo 7 u 8 (según edición) subidas por estudiantes. Explica elicitación, análisis y especificación de requisitos.
- **Idioma:** Español (la mayoría de resúmenes)
- **Nivel:** Introductorio a intermedio

## Levantamiento de Requisitos

### 4. Técnicas de Elicitación de Requisitos — artículo introductorio
- **URL:** https://www.oreilly.com/library/view/requirements-engineering-fundamentals/9781457199752/
- **Descripción:** Fundamentos de ingeniería de requisitos. El primer capítulo es gratuito en O'Reilly Preview. Cubre las técnicas principales: entrevista, cuestionario, observación y workshops.
- **Idioma:** Inglés
- **Nivel:** Referencia

### 5. Plantilla de Documento de Requisitos de Software (SRS) — IEEE 830
- **URL:** https://www.ieee.org/content/dam/ieee-org/ieee/web/org/about/whitepapers/
- **Descripción:** Estructura estándar de un documento de especificación de requisitos. Útil para ver en qué se basará el entregable S02 al crecer hacia la semana 7.
- **Idioma:** Inglés
- **Nivel:** Referencia para la propuesta técnica

## Colombia: Contexto de Requisitos en Proyectos TI

### 6. Guía de Requisitos para Proyectos TI — MinTIC
- **URL:** https://www.mintic.gov.co/arquitecturati/630/w3-propertyvalue-8118.html
- **Descripción:** El Marco de Referencia de Arquitectura TI de Colombia incluye una sección de requisitos para sistemas de información del Estado. Aplicable también a PYMES que quieran seguir buenas prácticas.
- **Idioma:** Español
- **Nivel:** Introductorio — contexto colombiano
