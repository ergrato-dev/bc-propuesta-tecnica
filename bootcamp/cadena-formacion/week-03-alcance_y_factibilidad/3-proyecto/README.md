## 📋 Descripción

Esta semana darás un paso crucial: definirás **qué SÍ** y **qué NO** entra en la primera versión de tu proyecto real. Tomarás los requisitos que documentaste en la Semana 2 y convertirás esa lista en una declaración de alcance formal.

Este documento es la **Sección 3 de tu propuesta técnica**. Se conecta directamente con los requisitos de S02 — no pueden entrar en el alcance funcionalidades que no tienen requisito documentado.

## ❓ Preguntas Guía

Antes de completar la plantilla, respóndete estas preguntas sobre tu proyecto:

### Sobre el Alcance
- ¿Cuáles son los 3 o 4 módulos principales que debe tener tu sistema?
- ¿Hay funcionalidades que el cliente pidió pero que son demasiado complejas o costosas para la versión 1?
- ¿Qué dejas para una "versión 2" o "fase posterior"?
- ¿Qué pasaría si alguien te pide algo que no acordaste construir? ¿Tienes documentado que está fuera del alcance?

### Sobre los Entregables
- ¿Cuáles son los módulos de software que entregarás?
- ¿Qué documentación producirá el proyecto (propuesta, manual, diagramas)?
- ¿Cómo sabrás (tú y tu cliente) que cada módulo está terminado?

### Sobre la Factibilidad
- ¿Conoces las tecnologías que vas a usar? ¿Necesitas aprender algo nuevo?
- ¿Tienes dónde alojar el sistema (hosting, servidor, nube)?
- ¿El cliente tiene los equipos y la conexión para usar el sistema?
- ¿El esfuerzo que requiere el proyecto está dentro de lo que puedes hacer en el tiempo disponible?
- ¿Los beneficios del sistema justifican la inversión?

## ✅ Criterios de Aprobación

El documento será evaluado según estos criterios:

| Criterio | Mínimo esperado |
|----------|----------------|
| Declaración de alcance (in scope) | Al menos 4 módulos o funcionalidades descritos con precisión |
| Out of scope | Al menos 3 ítems con justificación |
| Lista de entregables | Al menos 6 entregables (4 software + 2 documentación) |
| WBS básico | Al menos 3 módulos de nivel 1 con subcomponentes de nivel 2 |
| Factibilidad técnica | Evaluación de al menos 4 criterios con resultado y análisis |
| Factibilidad operativa | Evaluación de al menos 3 criterios |
| Restricciones y supuestos | Al menos 2 restricciones y 2 supuestos identificados |
| Formato y lenguaje | Documento completo, formal, sin secciones vacías sin responder |
