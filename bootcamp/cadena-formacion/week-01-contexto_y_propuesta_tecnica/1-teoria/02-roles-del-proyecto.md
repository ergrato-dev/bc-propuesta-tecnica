# 📖 02 — ¿Quiénes participan en un Proyecto?

> Teoría · Semana 1 · Cadena de Formación
> Aplica al caso de estudio: **FerreMax S.A.S.**

---

## 🎯 Objetivos

- Identificar los roles principales en un proyecto de software
- Distinguir entre cliente, usuario final y sponsor
- Reconocer responsabilidades del PM y del equipo de desarrollo
- Mapear los stakeholders del caso FerreMax

---

## 1. ¿Qué es un Stakeholder?

Un **stakeholder** (parte interesada) es cualquier persona o grupo que:

- **Afecta** al proyecto (toma decisiones, aprueba, financia), o
- **Es afectado** por el proyecto (su trabajo cambia cuando el sistema esté listo)

> ⚠️ Error común: creer que los stakeholders solo son el cliente y el equipo. Los empleados que van a usar el sistema también son stakeholders aunque no contraten nada.

---

## 2. Los 5 Roles Principales

### 🏢 Cliente / Sponsor

**¿Quién es?** La persona o empresa que paga y autoriza el proyecto.

**Responsabilidades:**
- Aprobar o rechazar la propuesta técnica
- Definir qué quiere (a grandes rasgos)
- Proveer el presupuesto
- Tomar decisiones cuando hay cambios importantes

**En FerreMax:** Carlos Mendoza, Gerente General. Él decide si el proyecto sigue adelante o no.

---

### 👤 Usuario Clave / Product Owner

**¿Quién es?** La persona que conoce el negocio por dentro y trabaja de cerca con el equipo de desarrollo.

**Responsabilidades:**
- Explicar cómo funciona el negocio hoy
- Validar que lo que se construye es lo que se necesita
- Priorizar qué debe hacerse primero

**En FerreMax:** Laura Gómez, Jefa de Bodega. Ella sabe exactamente cómo se mueve el inventario día a día.

---

### 🗂️ PM — Project Manager

**¿Quién es?** El responsable de que el proyecto llegue a buen término.

**Responsabilidades:**
- Planificar el proyecto (alcance, tiempo, costo)
- Coordinar al equipo
- Gestionar riesgos
- Comunicarse con el cliente
- **Elaborar la propuesta técnica**

**Nota:** En proyectos pequeños (como los del bootcamp), el mismo desarrollador senior puede hacer de PM.

---

### 💻 Equipo de Desarrollo

**¿Quiénes son?** Los profesionales que construyen el software.

**Roles comunes:**

| Rol | Qué hace |
|-----|---------|
| Desarrollador backend | Lógica del servidor, base de datos, APIs |
| Desarrollador frontend | Interfaz visual, experiencia de usuario |
| QA / Tester | Pruebas, detección de errores |
| DBA | Diseño y administración de la base de datos |

**En proyectos pequeños** (PYME colombiana) un mismo desarrollador puede cubrir varios de estos roles.

---

### 🔍 Consultor / Analista

**¿Quién es?** Especialista externo que ayuda a definir requisitos o a evaluar soluciones.

**Responsabilidades:**
- Levantar requisitos con el cliente
- Documentar procesos de negocio
- Recomendar tecnologías o arquitecturas

**Nota:** En el bootcamp, el aprendiz actúa como consultor cuando hace la propuesta técnica para el cliente.

---

## 4. ¿Por qué importa identificar los roles?

Porque cada persona tiene **expectativas diferentes** del proyecto:

- El **sponsor** quiere el proyecto terminado en tiempo y presupuesto
- El **usuario clave** quiere que el sistema sea fácil de usar
- El **equipo de desarrollo** necesita requisitos claros para no reprocesar
- El **consultor** necesita acceso a información del negocio

Si no identificas bien los roles desde el inicio, terminas entregando algo que satisface a uno pero frustra a los demás.
