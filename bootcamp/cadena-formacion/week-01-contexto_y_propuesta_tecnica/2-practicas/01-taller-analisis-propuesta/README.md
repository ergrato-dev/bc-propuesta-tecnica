# 🛠️ Taller 01 — Análisis de una Propuesta Técnica Real

> Práctica guiada · Semana 1 · Cadena de Formación
> Caso de estudio: **FerreMax S.A.S.**

---

## 🎯 Objetivo del Taller

Analizar una propuesta técnica simplificada y real para identificar sus secciones, comprender su estructura y practicar el vocabulario de gestión de proyectos.

**Al finalizar el taller podrás:**
- Reconocer las secciones de una propuesta técnica
- Clasificar requisitos como funcionales o no funcionales
- Identificar stakeholders en un documento real
- Usar correctamente el vocabulario del glosario de la semana

## 📋 Paso 2: Clasificar los Requisitos

La propuesta menciona 8 cosas que el sistema debe hacer o cómo debe comportarse. Clasifica cada una.

**Recordatorio:**
- **RF (Requisito Funcional):** qué debe *hacer* el sistema
- **RNF (Requisito No Funcional):** cómo debe *comportarse* (velocidad, seguridad, usabilidad, etc.)

**Ejemplo resuelto:**

| # | Descripción | Tipo |
|---|-------------|------|
| 1 | "El sistema debe permitir registrar productos con código, nombre y precio" | RF |
