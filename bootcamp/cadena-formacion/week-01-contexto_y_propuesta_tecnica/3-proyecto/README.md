## ¿Qué es este entregable?

Es la **primera sección de tu propuesta técnica**. A lo largo de las 9 semanas irás construyendo cada sección, y al final del bootcamp tendrás una propuesta técnica completa para **tu proyecto real**.

Esta semana describes el contexto: quién es el cliente, cuál es el problema y por qué vale la pena resolverlo.

---

## ❓ Preguntas Guía

Antes de abrir la plantilla, responde mentalmente (o en papel) estas preguntas:

- ¿Cuál es el problema que tu proyecto resuelve? (descríbelo sin usar términos técnicos)
- ¿Quién tiene ese problema? ¿Tiene nombre y cargo?
- ¿Por qué es importante resolverlo ahora? ¿Qué pasa si no se resuelve?
- ¿Qué tipo de solución de software propones? (no tienes que ser exacto aún)
- ¿Quiénes se van a ver afectados por el proyecto — para bien o para mal?

## ✅ Criterios de Evaluación

Tu entregable será evaluado según la rúbrica `rubrica-evaluacion.md`. Enfócate en:

- Describir el problema desde la perspectiva del **cliente** (no del desarrollador)
- Justificar el proyecto con argumentos de valor: ahorro, eficiencia, impacto
- Identificar mínimo 3 stakeholders con nombre, cargo y su relación con el proyecto
- Usar lenguaje formal en todo el documento
