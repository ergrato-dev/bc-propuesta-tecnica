## 🛠️ Herramientas para la Propuesta Específica de FerreMax

Para construir los entregables de esta semana sobre el caso FerreMax, se recomiendan:

### 1. Portada y Formato
- **Canva** → plantillas de "propuesta de servicios" → adaptar colores al cliente
- **Google Docs** → crear el documento base con estilos de título H1/H2/H3

### 2. Tablas (Módulos, Costos, Cronograma)
- **Google Sheets** o **LibreOffice Calc** para tablas complejas → copiar a Google Docs como imagen
- **Markdown** en GitHub → tablas `|col1|col2|` para la versión versionada del documento

### 3. Diagramas (Arquitectura, WBS)
- **Draw.io** → diagrama de bloques de la arquitectura del sistema FerreMax GI
- **Miro** → mapa mental del WBS de los 6 módulos (M1–M6)

### 4. Control de Versiones del Documento
- **GitHub** → repositorio del proyecto, subir la propuesta técnica como `propuesta-tecnica-v1.0.md`
- **Google Docs** → historial de versiones integrado (Ver → Historial de versiones)

## 💡 Buenas Prácticas al Citar Fuentes en la Propuesta

En una propuesta técnica profesional, es buena práctica referenciar los estándares que usaste:

```
Este proyecto se desarrollará bajo las directrices del PMI PMBOK (Project Management 
Body of Knowledge, 7a edición) para la gestión del alcance, cronograma y costos.
Los criterios de calidad del software se basan en la norma ISO/IEC 25010.
```

> ⚠️ Citar estándares da credibilidad a tu propuesta ante el cliente. No es obligatorio para todos los proyectos, pero sí es una señal de profesionalismo.
