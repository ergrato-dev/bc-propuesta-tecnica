# 🚀 Proyecto Semana 6 — Presupuesto de tu Proyecto

> **Versión**: Cadena de Formación · Tecnólogo ADSO · Semana 6 de 9

---

## 📋 Descripción

Esta semana construyes la **estimación de costos de tu proyecto real**. Este documento es una de las secciones más importantes de la propuesta técnica: el cliente toma decisiones de inversión basado en él.

El presupuesto se construye sobre los datos que ya tienes de semanas anteriores:
- **Semana 4**: días-persona por módulo → son la base del costo de recurso humano
- **Semana 5**: cronograma → confirma la duración (semanas) que defines como período de pago

Este documento es la **Sección 6 de tu propuesta técnica**. Al llegar a la Semana 9, lo incluirás directamente sin mayor modificación.

## 📝 Instrucciones

1. Abre el archivo `starter/entregable-s06-presupuesto.md`
2. Ten a la mano:
   - Tu `entregable-s04-estimacion.md` (días-persona por módulo)
   - Tu `entregable-s05-cronograma.md` (duración total)
3. Lee las instrucciones de cada sección antes de completarla
4. Consulta las tarifas del mercado en la Teoría 02 y elige una tarifa apropiada para tu perfil y el de tu equipo
5. Muestra los cálculos — no pongas solo el resultado final
6. Usa lenguaje formal — este es un documento que le mostrarías a un cliente

> ⚠️ **Importante**: Este documento NO debe contener información de FerreMax. Usa exclusivamente los datos de tu proyecto real aprobado.

---

## ❓ Preguntas Guía

Antes de abrir la plantilla, respóndete estas preguntas:

### Sobre el Recurso Humano
- ¿Cuántas personas trabajan en el proyecto? ¿Cuál es el perfil de cada una?
- ¿Eres tú el único desarrollador, o hay más personas en el equipo?
- ¿Qué tarifa es razonable para tu perfil actual? (Aprendiz / Junior / Semi-senior)
- ¿Los días-persona de tu S04 están bien distribuidos entre las personas del equipo?

### Sobre la Infraestructura
- ¿Tu sistema necesita un servidor? ¿Usarás hosting compartido, VPS o servicio cloud?
- ¿Necesitarás un dominio propio?
- ¿Usarás algún servicio de pago (SendGrid para emails, Cloudinary para imágenes, etc.)?

### Sobre las Licencias
- ¿Usas alguna herramienta de pago? ¿O tu stack es completamente open source?

### Sobre la Contingencia
- ¿Qué tan bien definido está el alcance de tu proyecto? (Ver tu S03)
- Con ese nivel de certeza, ¿usarías 10%, 15% o 20% de contingencia?

### Sobre el Margen
- ¿Tu propuesta es para un cliente real, para el instructor, o académica?
- Si es para un cliente real, ¿cuánto margen agregarías?

## ✅ Criterios de Aprobación

Tu entregable está completo si:

- [ ] Las secciones A (RH), B (Infraestructura), C (Licencias) y D (Contingencia) están presentes
- [ ] Los días-persona coinciden con tu `entregable-s04-estimacion.md`

- [ ] Incluyes al menos 3 supuestos del presupuesto
- [ ] El total final tiene sentido con la magnitud del proyecto
